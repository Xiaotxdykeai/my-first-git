# 第一章思维导图

## 1. 数据结构基本概念
- **数据**
- **数据元素、数据项**
- **数据对象、数据结构**
- **数据类型、抽象数据类型（ADT）**

## 2. 数据结构的三要素
### 2.1 逻辑结构
- **线性结构**（一对一）
  - 线性表、栈、队列、串等
- **非线性结构**
  - **集合**（无明确关系）
  - **树形结构**（一对多）
  - **图状结构 / 网状结构**（多对多）

### 2.2 物理结构（存储结构）
- **顺序存储**（连续存储）
- **非顺序存储**
  - **链式存储**
  - **索引存储**
  - **散列存储**

### 2.3 数据的运算
- 根据逻辑结构来定义基本运算
- 根据存储结构来实现运算
- 定义 ADT 即定义 **逻辑结构 + 运算**
- 确定存储结构后，才能实现该数据结构

## 3. 数据类型与抽象数据类型（ADT）
- **数据类型**：值的集合 + 一组操作
  - **原子类型**：值不可再分（如 int）
  - **结构类型**：值可分解为若干分量（如 struct）
- **抽象数据类型（ADT）**
  - 抽象数据组织及与之相关的操作
  - 定义 ADT = 定义一种数据结构（逻辑结构 + 运算）
  - 存储结构确定后，才能“实现”该 ADT

## 4. 算法
### 4.1 算法的基本概念
- **程序 = 数据结构 + 算法**
- 数据结构：要处理的信息
- 算法：处理信息的步骤

### 4.2 算法的五个特性
- **有穷性**：有穷时间内执行完（程序可以是无穷的）
- **确定性**：相同输入产生相同输出
- **可行性**：可用已有基本操作实现
- **输入**：零个或多个输入
- **输出**：至少一个输出

### 4.3 “好”算法的特质
- **正确性**：能正确解决问题
- **可读性**：描述清晰，易于理解
- **健壮性**：能处理异常情况
- **高效率与低存储量需求**（时间/空间复杂度低）

## 5. 时间复杂度
### 5.1 如何计算
1. 找一个基本操作（最深层循环）
2. 分析执行次数 `x` 与问题规模 `n` 的关系 `x = f(n)`
3. 取数量级 `O(x)` 即为时间复杂度 `T(n)`

### 5.2 常用技巧
- **加法规则**：`O(f(n)) + O(g(n)) = O(max(f(n), g(n)))`
- **乘法规则**：`O(f(n)) × O(g(n)) = O(f(n) × g(n))`

### 5.3 常见复杂度阶（“常对幂指阶”）
`O(1) < O(log₂n) < O(n) < O(nlog₂n) < O(n²) < O(n³) < O(2ⁿ) < O(n!) < O(nⁿ)`

### 5.4 三种复杂度
- **最坏时间复杂度**：输入数据最坏情况
- **平均时间复杂度**：所有输入等概率出现时的期望
- **最好时间复杂度**：输入数据最好情况

## 6. 空间复杂度
### 6.1 普通程序
1. 找到所占空间大小与问题规模相关的变量
2. 分析所占空间 `x` 与 `n` 的关系 `x = f(n)`
3. `x` 的数量级 `O(x)` 即为空间复杂度 `S(n)`

### 6.2 递归程序
1. 找到递归调用深度 `x` 与 `n` 的关系 `x = f(n)`
2. `x` 的数量级 `O(x)` 即为空间复杂度 `S(n)`

### 6.3 常用技巧（同时间复杂度）
- 加法规则、乘法规则
- “常对幂指阶”同样适用
